Três métricas distintas — nenhuma está errada. CR Total = todos os cancelamentos ÷ base anterior (inclui encerramento, venda cancelada, etc.). CR Ajustado = apenas Churn=Sim ÷ base anterior (perda real de cliente por problema de produto/serviço). Taxa Não-Churn = apenas Churn=Não ÷ base anterior (cancelamentos administrativos). CR Total = CR Ajustado + Taxa Não-Churn. 2020 usa a própria base por não haver base de 2019.

O que é um Cliente Retido?
Um cliente retido é aquele que iniciou o processo de cancelamento mas foi convencido a permanecer pela equipe de suporte ou comercial. Diferente de um cancelamento normal (que gera perda de MRR), a retenção preserva a receita e demonstra capacidade de relacionamento pós-venda.

Impacto financeiro: Reter um cliente custa em média 5× menos do que adquirir um novo. O MRR recuperado pelos 7 retidos representa receita que teria sido perdida e que agora permanece na base ativa.

Como atualizar a base ativa:
  1. Exporte o relatório de posição de contratos do sistema como planilha
  2. No Google Sheets, abra a aba Base_Ativa e cole os dados (substitua tudo)
  3. Publique a aba como CSV: Arquivo → Publicar na web → Base_Ativa → CSV
  4. Cole o link em SHEETS_URL_ATIVOS no HTML (só precisa fazer isso uma vez)
  5. Clique ↻ Atualizar dados — a Curva ABC e KPIs recalculam sozinhos
